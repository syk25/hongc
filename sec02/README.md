# 섹션2. C 언어 소개
## 2.1 C 의 해부학
프로그램은 특정 목적을 위한 기능 수행이다. 
프로그램이 작동하기 위해서는 cpu와 메모리의 협력이 필요하다. 
메모리는 cpu에게 데이터를 전달하고 cpu는 기능을 수행해서 데이터를 가공한 후 다시 메모리에게 그 데이터를 보낸다.

기능은 곧 함수다. 함수란 데이터를 입력 받아 로직에 따라 가공한 후 그 결과물을 출력하는 것이다. 즉, 함수란 입력을 받아 출력을 하는 것이다.

프로그램은 함수의 조합체다. 함수들의 관계를 분석할 수 있으면 프로그램을 분석할 수 있다.

## 2.2 문장(statement)
```C
#include <stdio.h> // 전처리기 - 컴파일 전단계에서 참조 - 유명 문구 복붙

int main(void) // 메인함수 정의 시작
{ // 영역 시작

    // 변수 선언
    int a;
    int b;
    int c;

    // 변수에 값 대입
    a = 1;
    b = 2;

    // 변수에 연산 결과 대입
    c = a + b;

    // 함수 호출 - call or invoke
    printf("Result is %i", c);

    return 0; // 결과값을 반환
} // 영역 끝
```
프로그램에서 문장이란 작업을 수행하는 일련의 지시어다. 


## 2.3 변수가 필요한 이유
### 값과 변수
값이란 구체적인 데이터다. 변수는 값을 확장한 개념이다. 
```
1 + 2 = 3  // 값으로 표현
a + b + c  // 변수로 표현
```
변수는 연산과정에서 값을 일반화할 때 사용할 수 있다.

### CPU와 RAM, 변수
CPU와 RAM은 데이터의 주소를 통해 의사소통한다. 하지만 개발할 때 주소로 데이터를 접근하는 것은 매우 번거롭다. C에서는 주소 대신에 변수 개념을 도입해서 데이터에 접근할 수 있게 한다.

## 2.4 자료형이 필요한 이유
```
자료형은 변수에 담길 데이터의 종류를 지정한다.
```

```C
int a = 10;
float x = 10.2f;
char b = 'A';
```
CPU는 주어진 데이터에 따라 다른 방식으로 연산한다. 따라서 CPU에게 변수에 담긴 데이터의 종류가 무엇인지 미리 알려줘야 한다.

데이터의 종류마다 필요한 메모리 공간도 다르다. 

## 2.5 변수를 선언하는 방법
변수란 메모리 주소에 붙인 이름이다. 변수를 선언한다는 것은 메모리를 이름으로 접근하겠다는 의미이다. 
```C
int a; // 메모리에 int의 크기 만큼의 공간을 할당하고 a라는 이름을 붙임. 10이 들어가 있음
```

### 변수이름 명명규칙
```
1. 변수 이름 사이에 빈칸 대신에 언더바 사용
2. 변수의 시작을 숫자로 해서는 안됨
3. 같은 영역 안에 동일한 이름의 변수가 2개 이산 존재 불가
4. 예약어는 변수의 이름으로 쓸 수 없음
```

변수는 선언하는 것과 동시에 초기화할 수 있다.
```C
int a = 10;
```

## 2.6 printf() 사용법
함수는 입력을 받아 출력을 내보낸다. printf()는 데이터를 전달하면 콘솔 창에 데이터가 표시된다.

문자열을 콘솔 창에 보여주고 싶은 방식에 따라 printf()의 쓰임새가 달라진다.

### 용도1: 문자열 출력하기

```C
printf("문자열 출력하기");
```

### 용도2: 문자열 출력 + 줄 바꾸기
```C
printf("문자열 출력하기.\n");
```
문자열 뒤에 새 줄을 만들고 싶으면 escape 문자 \n을 사용해야한다.

### 용도3: 숫자를  출력하기
```C
int x = 1;
int y = 2;
int z = x + y;
printf("%d + %d = %d", x, y, z);
```
문자열 내에 변수에 따라 숫자를 다르게 출력해야하는 경우에 형식지정자를 사용할 수 있다.

이외에 다양한 형식으로 문자열을 출력할 수 있다.

## 2.9 함수 만드는 법
```
1. 함수 선언하기
    1) 입력으로 쓸 매개변수 정하기
    2) 출력 형태인 반환형 정하기
    3) 함수의 이름 정하기
2. 함수 정의하기
    1) 내부 로직을 구현하기
```
함수도 변수처럼 사용하기 전에 선언되어야 한다. 하지만 코드가 복잡해지는 것을 막기 위해 C에서는 prototype을 사용한다. prototype은 함수의 선언과 정의를 분리한다. 
```C
#include <stdio.h> 

void say_bye(void); // prototype, declaration

int main(void)
{

    say_bye(); // 함수 사용

}


void say_bye(void) // definition
{
    printf("Bye World\n");
}
```

## 2.10 디버거 사용법
### 개념
디버거는 소프트웨어 개발 시 버그를 찾고 수정하는 데에 쓰는 프로그램이다. 

### 종류
디버거는 개발환경에 따라 종류가 다양하다. C 의 경우 GDB를 많이 쓴다. 통합개발환경과 함께 묶어서 제공되는 경우가 있지만 필요에 따라 디버거를 다른 걸로 쓸 수 있다. VSCode의 경우 별도의 extension을 설치해야한다.


### 용어
#### 상태(state)
프로그램 실행 시 변수의 값이다. 각 행이 실행 될 때마다 변수에 어떤 처리를 했냐에 따라 변수들의 상태가 바뀔 수 있다. 

#### 중단점(breaking point)
중단점이란 디버깅 시 프로그램이 실행 중단되는 시점이다. 중단점부터 디버거를 통해 행을 하나씩 실행할 수 있다. 중단점은 여러개 설정할 수 있다.

#### step over
이번 행이 함수인 경우 함수 내부로 들어가지 않고 실행만하고 상태를 확인하는 행위다. 함수가 아닌 경우에는 해당 행을 실행하고 관련 된 변수들의 상태를 변경한다.

#### step into
다음 행에 함수가 있고 내부에 들어갈 수 있는 경우에 함수 내부의 행을 실행하는 행위다. 다음 행이 함수가 아닌 경우에 step over와 동일히다. 내부에 들어갈 수 없는 경우에 함수를 실행만 하고 그에 따른 상태변화만 확인할 수 있다.

### 사용법
```
1. 중단점 설정
2. step into를 통해 행 하나씩 실행하기
3. 변수들의 상태 확인하기
```

## 2.11 문법오류와 문맥오류
프로그램은 문법오류, 문맥오류, 의미오류가 나타날 수 있다.

### 문법오류(Syntax errors)
문법오류는 코드가 프로그래밍 언어의 문법을 따르지 않을 때 발생하는 오류다. 문법오류는 컴파일 단계에서 발견된다. 고치기 쉬운 오류다.

### 문맥오류(contextual errors, runtime errors)
문맥오류는 프로그램의 문법이 올바르지만 실행 중에 예산치 못한 상황이 발생하여 프로그램이 중단되는 경우이다. 존재하지 않는 파일 열기, 메모리 접근 오류, 0으로 나누기 등이 문맥 오류를 일으킨다.

### 의미오류, 논리오류(Semantic error, logical error)
코드의 의미가 프로그램의 의도와 일치하지 않을 때 발생하는 오류가 의미오류이다. 논리오류라고도 한다. 변수 이름, 함수 호출, 알고리즘 구현이 프로그램의 요구사항이나 로직과 맞지 않을 때 발생하는 오류다.


## 2.12 읽기 좋은 코드를 만드는 요령
```
1. 시작과 끝을 명확하게 표시하기
2. body 부분에서 indentation 적절하게 사용하기
3. 좋은 코드를 많이 보기
```
좋은 코드란 구조화가 되어 있고 의미를 이해하는 데에 별다른 노력이 들어가지 않는 코드다. 경우에 따라서는 주석을 활용해서 의미를 전달해야한다.
